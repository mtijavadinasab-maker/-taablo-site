# تابلو — داشبورد تحلیل اقتصادی

نسخه‌ی ۰.۱ — فاز اول (داشبورد داده + نمودار روند + سیگنال‌های نمونه)

## این نسخه چیست و چی نیست

**هست:** یک داشبورد وب کاملاً کارکن که نرخ چند ارز جهانی را به‌صورت زنده می‌گیرد، یک نمودار روند رسم می‌کند، و بخش «سیگنال‌ها»/«سناریو» را با ساختار نهایی (ولی داده نمونه) نمایش می‌دهد.

**نیست:** یک سیستم پیش‌بینی هوش مصنوعی. فعلاً هیچ مدل پیش‌بینی واقعی پشت سناریوها نیست — این عمداً است، چون ساخت یک مدل پیش‌بینی معتبر نیاز به داده تاریخی زیاد و اعتبارسنجی دارد که در فازهای بعدی اضافه می‌شود.

## ساختار فایل‌ها

```
econ-site/
├── index.html   ساختار صفحه
├── style.css    طراحی بصری (تم تیره، الهام از تابلوی بورس)
├── script.js    منطق: گرفتن داده، رسم نمودار، نمایش سیگنال/سناریو
└── README.md    همین فایل
```

## چطور همین الان ببینمش؟

فایل `index.html` را در هر مرورگری (روی گوشی یا کامپیوتر) باز کن. چون از CDN برای Chart.js استفاده می‌کند، نیاز به اینترنت دارد.

## چطور رایگان آنلاینش کنم؟

ساده‌ترین راه، **GitHub Pages** یا **Netlify Drop** است:
1. یک حساب رایگان در github.com بساز
2. یک ریپازیتوری جدید بساز و این ۳ فایل (html/css/js) را آپلود کن
3. در تنظیمات ریپو، بخش Pages را فعال کن — سایتت آنلاین می‌شود، رایگان، با یک لینک عمومی

(اگه بخوای، مرحله به مرحله همین کار رو با هم انجام می‌دیم.)

## نقشه راه واقعی (برای پروژه ۳ ساله)

### فاز ۱ — همین نسخه ✅
داشبورد داده + ساختار UI نهایی با داده نمونه.

### فاز ۲ — داده واقعی ایران
- جایگزینی API نرخ رسمی با یک منبع نرخ بازار آزاد (نیاز به یک API پولی مثل navasan.tech یا ساخت یک بک‌اند که داده را از منابع مجاز جمع‌آوری کند)
- این بخش نیاز به یک برنامه‌نویس بک‌اند دارد (Python/Node) چون داده باید در یک سرور ذخیره و تاریخی شود؛ مرورگر به‌تنهایی کافی نیست.

### فاز ۳ — سیگنال‌های واقعی
- اتصال به یک فید خبری واقعی (RSS اقتصادی) از طریق یک سرور واسط (به‌دلیل محدودیت CORS، این کار مستقیم از مرورگر ممکن نیست)
- طبقه‌بندی خبر به «مثبت/منفی/خنثی» — می‌تواند دستی شروع شود (خودت به عنوان تحلیلگر طبقه‌بندی کنی) و بعداً خودکار شود

### فاز ۴ — مدل سناریو
- استفاده از مدل‌های آماری ساده (رگرسیون، همبستگی) روی داده تاریخی جمع‌آوری‌شده
- نمایش شفاف درصد عدم‌قطعیت (هیچ‌وقت ادعای «پیش‌بینی قطعی» نکن — این هم از نظر علمی درست‌تر است هم برای رزومه معتبرتر)

## نکته برای رزومه

مستندسازی همین مسیر (چرا فاز ۱ را اینطور طراحی کردی، چرا داده نمونه، چه تصمیماتی گرفتی) خودش بخشی از ارزش پروژه است — نشون می‌ده که مدیریت محصول کردی، نه فقط کد گرفتی.
